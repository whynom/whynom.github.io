2025-04-10 · 2min

I added something

I recently released my first github repo which is two functions that wrap around a some keychain functions so that someone can store a string on a users iOS (should be any Apple device) and save that information between user installations. My impetus being, I wanted to make it so I can have someone use my app once, and then have to pay to get more uses. I didn't want them to be able to get the free use, and then uninstall and reinstall and get another free use.

The app is extremely basic. It's basically doing a math problem, and giving a pretty chart or two to show the output of the math. The suggestion from my buddy, LLM, was to set up a server that tracked users' use, which was entirely way too much work for such a simple app. A reason I like developing for Apple devices is they give me a lot, and I didn't want to go down the rabbit hole of setting up and mainting a server. The server would most likely be more work than the app.

The one pitfall of useing the keychain to track usage is that if someone installs my app on their iPhone, and then on their iPad, they do, indeed, get another free use. Although not ideal, if you're so gung-ho on cheating me out of $2, I think you deserve the win!
